# DevEnv Manager v1.0.0

开发环境管理工具（Java / Node / Python / Go / MySQL / Redis / Maven / Gradle / Nginx / Docker 等一键安装、多版本切换、环境变量管理）。

## 本版本修复与改进

- **数据目录重构（根治重装丢数据）**：运行时状态从「安装目录内」改到 `AppData\Roaming\DevEnv Manager`，重装 / 覆盖安装天然保留已配置的根目录与已装工具清单。
- **修复「开始配置」报错 ENOTDIR**：生产环境 `app.getAppPath()` 指向 `app.asar` 文件、在其内建目录会抛 ENOTDIR 的问题已解决。
- **修复启动黑屏**：窗口改为 `ready-to-show` 后才显示，杜绝启动期黑屏。
- **彻底杜绝旧版遗留根目录 `E:\a` 被重建 / 使用**：
  - `config:setRoot` 直接拒绝 `E:\a`；
  - `loadRoot` 读到残留 `E:\a` 视为未配置，绝不重建；
  - 迁移逻辑碰到 `E:\a` 直接跳过，并**主动清掉安装目录内那份指向 `E:\a` 的死 `devenv-data`**，升级时不会被 NSIS 原样搬回。
  - 结论：新版在任何情况下都不会在磁盘上创建 `E:\a`；历史残留的 `E:\a` 文件夹可手动删除，不影响本软件运行。
- **覆盖安装（同 appId `com.devenv.manager`）**：不卸载旧版即可直接安装，已装工具不丢失。
- **未提权安装前弹「用户级变量」提示**：双击打开（未以管理员运行）点击「开始安装」时，会先弹窗说明本次环境变量将写入**用户级(HKCU)**（仅当前账户生效、免管理员），并提示若需写入系统级(HKLM)请右键「以管理员身份运行」；可勾选「不再提示」关闭后续弹窗。提权运行时则直接写入系统级、不弹此提示。

## 使用提示

- 首次启动请在「设置中心」指定一个**位于程序安装目录之外**的安装根目录（如 `D:\devenv-root`），这样重装 / 升级时已有工具不会丢失。
- 若磁盘上仍存在旧版遗留的 `E:\a` 文件夹，直接删除即可，与新版无关。
- **环境变量写入与管理员**：默认双击打开即可用——未以管理员运行时，开发工具的 `HOME` / `PATH` 自动写入**用户级(HKCU，仅当前用户生效、免管理员)**；右键「以管理员身份运行」时则写入**系统级(HKLM，对所有用户生效)**。两种情况下工具文件都照常安装，不会因权限问题静默失败或报错。
- **关于代码签名**：本安装包当前**未做代码签名**（构建环境无代码签名证书）。首次运行时 Windows SmartScreen 可能提示「未知发布者」，属正常现象——点击「仍要运行」即可；如需消除该提示，可在本机用自有证书对 `DevEnv Manager-1.0.0-setup.exe` 签名后再分发。
